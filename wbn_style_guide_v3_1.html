Ghost Compatibility Rules
Mandatory
Rule 1 — IIFE Wrapping
All widget JS wrapped in (function(){ 'use strict'; ... })(); — prevents global scope collisions between widgets on the same Ghost page.
Rule 2 — No Inline onclick=
Never use inline onclick= handlers. Ghost's CSP blocks them silently. All binding via addEventListener. Dynamic lists use event delegation on the parent container.
Rule 3 — Self-Contained
Every widget is fully self-contained within its wrapper div. No bleed, no shared CSS or JS. Copy-paste deployable into any Ghost HTML block.
Rule 4 — localStorage Persistence
All editable data saved to localStorage. Always wrap read/write in try/catch. Provide Download Data (export JSON) and Upload Data (import JSON) in admin.
Rule 5 — Admin Access Pattern
Password gate on all admin panels. Login/logout state to localStorage. Admin hidden by default. Enter key via keydown listener. Password as plain constant — change per deployment.
Rule 6 — Sponsor / Advertising Widget
Every platform widget includes a sponsor section. Default: generic "Sponsor" buttons. Admin-editable, persisted to localStorage.
Rule 7 — Version Stamp
Footer: WBN WidgetName™ v1.0 | MM-DD-YY H:MM pm PST — Source Code Pro · 11–12px · color: var(--muted).
Rule 8 — Responsive
All widgets single-column on mobile. Grid layouts collapse at max-width: 1080px (wide) or 760px (standard). No horizontal scroll.
Do & Don't
Do
Use Robin Blue (#59c2eb) as the primary interactive accent — buttons, active states, borders.
Do
Use contrast-safe status colours — #13795b success, #9a6700 warning, #a61b1b danger — as text on white.
Do
Use Arial as the default UI and body font. Universal, no external load.
Do
Use the frosted panel header: rgba(255,255,255,.58) with border-bottom on all panels.
Do
Use chips for inline card metadata (partner, category, tools). Use pills for topbar status indicators.
Do
Wrap all JS in an IIFE and bind all events via addEventListener.
Don't
Don't use the old lighter success/warning/danger colours (#1f9d63, #e6a23c, #d9534f) as text on white — they fail contrast. They are retired in v3.1.
Don't
Don't use inline onclick= handlers. They fail silently in Ghost HTML blocks.
Don't
Don't use sharp corners — the rounded standard is the WBN design language.
Don't
Don't put body copy in Oswald — display/heading use only.
Don't
Don't use Libre Franklin — retired in v3.0. Arial handles all body copy.
Don't
Don't use dark backgrounds — the light gradient shell is the WBN standard.
