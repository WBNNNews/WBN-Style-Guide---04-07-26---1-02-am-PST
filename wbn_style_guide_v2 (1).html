Do & Don't
Do
Use Robin Blue (#59c2eb) as the primary interactive accent — buttons, active states, borders.
Do
Use Arial as the default UI and body font — it is universal and requires no external load.
Do
Use Source Code Pro for timestamps, metadata, hex codes, ticker text, and version stamps.
Do
Use border-radius: 10px on buttons and 14–18px on panels for a clean, modern broadcast feel.
Do
Keep card and panel surfaces white (#ffffff) with the blue gradient for the shell/outer wrapper.
Do
Use event delegation for dynamically rendered admin buttons — not inline onclick attributes.
Do
Wrap all widget JS in an IIFE (immediately invoked function expression) to avoid global scope conflicts.
Don't
Don't use inline onclick= handlers for dynamically rendered elements — they break in Ghost HTML blocks.
Don't
Don't use red as a primary brand colour — it is reserved for danger/delete states only.
Don't
Don't spread localStorage reads into multiple calls — batch related data under a single key where possible.
Don't
Don't mix font weights carelessly — headings/labels use 700, body uses 400–500 only.
Don't
Don't put body copy in Oswald — it is a display/heading font only, used sparingly.
Don't
Don't use dark/black backgrounds on the main page — the light gradient theme is the WBN standard.

Build Engine Checklist

Every widget deployed to a WBN platform must satisfy these requirements before going live.

✓ Fully self-contained — no bleed outside the wrapper div
✓ Ghost-compatible — clean HTML/CSS/JS, no framework dependencies
✓ Admin panel with working login and logout
✓ localStorage-backed with export/import (Download Data / Upload Data)
✓ Sponsor / Advertising widget included
✓ Version stamp and timestamp in footer
✓ Responsive — single-column on mobile
✓ All JS wrapped in IIFE — no global namespace pollution
✓ Dynamic buttons use event delegation — no inline onclick=
